# Estrategia de Producto

## Vision

Crear un CRM moderno para pequenas y medianas empresas que atienden, venden y dan seguimiento por WhatsApp.

El producto debe ayudar a responder mas rapido, ordenar clientes, cerrar mas ventas y reducir trabajo manual sin volverse complicado. La experiencia principal debe sentirse como WhatsApp con superpoderes de CRM, no como un sistema pesado de administracion.

## Principio Permanente

Antes de crear cualquier nueva pantalla o funcionalidad, analizar si ya existe una forma mas simple de resolver el mismo problema.

Priorizar siempre simplicidad, velocidad e impacto en ventas sobre cantidad de funciones. Cada nueva caracteristica debe aumentar el valor del producto sin hacerlo mas complejo de usar.

Si una funcion no ahorra tiempo, no reduce errores, no mejora seguimiento o no ayuda a vender mas, no debe agregarse.

## Preguntas Antes De Implementar

Toda nueva funcion debe responder claramente:

- Que problema resuelve.
- Como ahorra tiempo.
- Como ayuda a vender mas.
- Como reduce errores.
- Como se podria resolver con menos clics.
- Como lo resolverian productos lideres como HubSpot, GoHighLevel, Kommo, Zoho CRM o Salesforce, sin copiar su interfaz.

## Usuario Objetivo

El sistema debe servir para negocios que venden productos o servicios por WhatsApp:

- Constructoras.
- Inmobiliarias.
- Restaurantes.
- Clinicas.
- Dentistas.
- Abogados.
- Arquitectos.
- Talleres.
- Esteticas.
- Gimnasios.
- Agencias.
- Escuelas.
- Empresas de limpieza.
- Servicios tecnicos.
- Carpinterias y fabricantes a medida.

El usuario no debe necesitar entrenamiento largo. La meta es que pueda entender lo esencial en menos de una hora.

## Criterios De UX

Cada funcion debe cumplir:

- Pocos clics.
- Visual y facil de entender.
- Rapida en escritorio y movil.
- Compatible con modo oscuro en futuras etapas.
- Sin formularios largos cuando pueda resolverse con datos progresivos.
- Sin configuraciones dificiles para tareas comunes.
- Estados claros: pendiente, enviado, respondido, pagado, atrasado, finalizado.
- Acciones cerca del contexto donde se necesitan.

## Filosofia De Producto

El sistema debe evitar convertirse en un menu infinito.

La mejor funcion es la que elimina trabajo:

- Detectar automaticamente.
- Sugerir en lugar de obligar.
- Completar datos desde la conversacion.
- Mover etapas con acciones naturales.
- Recordar pendientes antes de que se pierdan.

Cuando haya duda, elegir la opcion mas simple y reversible.

- WhatsApp multi cuenta.

- Modo IA y modo Humano por conversacion.
- Respuestas rapidas.
- Archivos, imagenes, audios, videos y documentos.
- Ficha del cliente.
- Etiquetas.
- Embudo.
- Cotizaciones.
- Cobros.
- Recordatorios.
- Transmision segmentada.
- Busqueda de chats.
- Sincronizacion de nombres, fotos y alias de contactos.

Antes de agregar modulos grandes, este nucleo debe sentirse confiable.

## Roadmap Recomendado

### Fase 1: CRM Comercial Simple

Objetivo: que ningun cliente se pierda.

Prioridad alta:

- Cliente unico sin duplicados.
- Historial completo por cliente.
- Notas internas.
- Archivos por cliente.
- Estado del cliente.
- Proximo seguimiento.
- Responsable del cliente.
- Vista de clientes sin seguimiento.
- Dashboard simple de pendientes.

Resultado esperado: el negocio sabe a quien responder, quien esta pendiente y que cliente puede cerrar venta.

### Fase 2: Automatizaciones De Seguimiento

Objetivo: vender mas sin depender de memoria manual.

Automatizaciones iniciales:

- Cuando llega un cliente nuevo.
- Cuando pasan X horas sin responder.
- Cuando se envia una cotizacion.
- Cuando el cliente acepta.
- Cuando paga.
- Cuando termina el servicio.
- Cuando pasan 30 dias.
- Cuando pasan 6 meses.
- Solicitar resena.
- Solicitar fotografia del trabajo.
- Enviar promocion futura.

Regla: cada automatizacion debe poder pausarse y entenderse facilmente.

### Fase 3: IA Operativa

Objetivo: que la IA ayude al equipo sin quitar control.

Funciones recomendadas:

- Responder automaticamente cuando el modo IA este activo.
- Sugerir respuestas cuando el modo Humano este activo.
- Resumir conversaciones largas.
- Detectar intencion del cliente.
- Extraer nombre, ciudad, proyecto, presupuesto, medidas y fecha tentativa.
- Clasificar clientes automaticamente.
- Generar cotizaciones desde el chat.
- Generar mensajes de seguimiento.

Regla: en chats importantes, la IA debe poder trabajar en modo revision antes de enviar.

### Fase 4: Ventas, Agenda Y Operacion

Objetivo: conectar ventas con ejecucion.

Funciones:

- Agenda de visitas.
- Instalaciones.
- Llamadas.
- Google Calendar.
- Outlook.
- Cotizaciones con versiones.
- Aceptar cotizacion desde WhatsApp.
- Convertir cotizacion en venta.
- Contratos.
- Firma.
- Adjuntar INE.
- Adjuntar comprobante.

### Fase 5: Finanzas Y Reportes

Objetivo: saber si el negocio esta ganando dinero.

Funciones:

- Pagos.
- Anticipos.
- Pendientes.
- Comisiones.
- Gastos.
- Utilidad.
- Caja.
- Ventas del dia.
- Tiempo promedio de respuesta.
- Clientes nuevos.
- Clientes perdidos.
- Embudo.
- Proximas instalaciones.
- Clientes sin seguimiento.

### Fase 6: SaaS Multiempresa

Objetivo: escalar el producto a varios negocios y equipos.

Funciones:

- Usuarios.
- Roles.
- Permisos.
- Sucursales.
- Multiples empresas.
- Multiples negocios.
- API.
- Webhooks.
- Respaldos.
- Logs.

## Reglas Para Nuevas Pantallas

Crear una nueva pantalla solo si:

- La accion no cabe naturalmente dentro del chat.
- El usuario necesita comparar muchos datos.
- Se necesita revisar una lista o calendario.
- La funcion se usara con frecuencia.

Si la accion puede resolverse con un modal, panel lateral o boton dentro del chat, evitar crear una pantalla nueva.

## Reglas Para Automatizaciones

Toda automatizacion debe mostrar:

- Que la activa.
- A quien aplica.
- Que mensaje o accion ejecuta.
- Cuando se ejecuta.
- Como pausarla.
- Historial de ejecuciones.

Nunca debe ser una caja negra.

## Reglas Para IA

La IA debe:

- Responder con contexto del negocio activo.
- Respetar el estado del cliente.
- No reiniciar la conversacion si ya existe historial.
- No inventar precios, fechas o promesas.
- Pedir datos faltantes de forma breve.
- Pasar a humano cuando el caso sea delicado.
- Evitar respuestas repetidas.
- Mantener tono profesional y natural.

## Medicion De Exito

El producto mejora si:

- Baja el tiempo promedio de respuesta.
- Suben los clientes con seguimiento.
- Bajan los chats olvidados.
- Suben cotizaciones enviadas.
- Suben pagos o anticipos.
- Baja la duplicacion de contactos.
- El usuario hace menos trabajo manual.

## Decision De Producto

Cuando haya varias opciones, elegir la que:

1. Mantenga WhatsApp como centro de trabajo.
2. Ahorre mas tiempo.
3. Ayude mas al cierre de venta.
4. Sea mas facil de entender.
5. Tenga menos riesgo tecnico.
